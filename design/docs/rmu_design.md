## 2 简介

复位管理单元（RMU）是芯片的复位控制中枢，负责汇总外部引脚复位（pin_rst_n）和模拟域上电复位（por_rst_n），并为各功能模块提供独立的软件复位信号。pin_rst_n 输入带有 500μs 数字滤波功能，可有效滤除外部引脚上的毛刺干扰。软件复位信号通过 AHB 总线配置 RMU 寄存器实现，支持对单个模块进行独立复位而不影响其他模块。

---

## 3 特点

- 双复位源输入：pin_rst_n（引脚复位）和 por_rst_n（上电复位）
- pin_rst_n 500μs 数字滤波：滤除小于 500μs 的脉冲干扰，确保复位可靠性
- 各模块独立软复位：uart_soft_rst_n、dsp_soft_rst_n、timer_soft_rst_n、sram_soft_rst_n、bfm_soft_rst_n
- AHB 从机接口：支持通过 AHB 总线读写软复位寄存器
- 异步复位、同步释放：所有复位输出经双触发器同步释放，避免亚稳态
- 复位状态可读：可查询当前软复位状态

---

## 4 基本原理

RMU 接收两组外部复位信号（pin_rst_n 和 por_rst_n），其中 pin_rst_n 先经过 500μs 数字消抖滤波器，滤除短脉冲干扰后与 por_rst_n 共同参与复位逻辑运算，产生全局复位。同时，通过 AHB 从机接口接收总线写入的软复位配置，为各模块产生独立的复位信号。

每个模块的最终复位信号 = (pin_rst_n_filtered AND por_rst_n AND 模块软件复位)。

软件复位采用"写 0 复位、写 1 释放"策略，上电默认为释放状态（1'b1）。

## 7 时钟和复位

- **时钟域**: hclk 域（通常 50MHz）
- **自身复位**: RMU 自身在 pin_rst_n（滤波后）和 por_rst_n 均为高后才开始工作
- **复位释放同步**: 所有复位输出经双触发器同步释放，避免亚稳态

## 8 功能描述

### 8.1 pin_rst_n 500μs 数字滤波

外部引脚复位信号 pin_rst_n 可能受到噪声干扰，因此 RMU 内部集成数字消抖滤波器：

- **滤波原理**: 当 pin_rst_n 出现跳变（高→低或低→高）时，启动计数器
- **滤波窗口**: 500μs（即 hclk 周期 × 滤波计数值）
  - 50MHz 时钟：500μs / 20ns = 25,000 个时钟周期
  - 16MHz 时钟：500μs / 62.5ns = 8,000 个时钟周期
- **判定规则**:
  - 若信号在滤波窗口内始终保持新状态，则认为该跳变有效
  - 若信号在滤波窗口内恢复原状态，则判定为毛刺，忽略该跳变
- **滤波参数**: 滤波阈值通过参数 `PIN_FILTER_CYCLES` 设定，默认 25000（对应 50MHz 下的 500μs）

```
时序示意：
  pin_rst_n    ─┐      ┌──────────────────
                └──────┘  (下降沿)
  filter_cnt   ────┬─────┬─────┬──────────
                   │ 递增 │ 递增 │ 达到阈值
                   v      v      v
  pin_filtered  ──────────────────┐      ┌──
                                  └──────┘  (延迟500μs后有效)
```

### 8.2 复位合并与同步释放

```
  pin_filtered_n ──┐
                   ├──► (AND) ──► sys_rst_n_raw
  por_rst_n ───────┘

  模块复位 = sys_rst_n_raw AND 模块软件复位信号
```

所有复位输出均经过双触发器同步释放：

```verilog
  // 同步释放（异步复位、同步释放）
  always @(posedge hclk or negedge rst_async) begin
      if (!rst_async) begin
          rst_sync1 <= 1'b0;
          rst_sync2 <= 1'b0;
      end else begin
          rst_sync1 <= 1'b1;
          rst_sync2 <= rst_sync1;
      end
  end
```

## 9 寄存器描述

### 9.1 寄存器列表

| 偏移地址 | 寄存器名 | 属性 | 复位值 | 说明 |
|----------|----------|------|--------|------|
| 0x00 | RMU_SRST | RW | 0x0000_001F | 软件复位控制寄存器 |
| 0x04 | RMU_STATUS | RO | 0x0000_0000 | 复位状态寄存器 |

### 9.2 RMU_SRST（0x00）— 软件复位控制寄存器

| 位 | 属性 | 复位值 | 说明 |
|----|------|--------|------|
| [0] | RW | 1'b1 | uart_soft_rst_n：1=释放，0=复位 |
| [1] | RW | 1'b1 | dsp_soft_rst_n：1=释放，0=复位 |
| [2] | RW | 1'b1 | timer_soft_rst_n：1=释放，0=复位 |
| [3] | RW | 1'b1 | sram_soft_rst_n：1=释放，0=复位 |
| [4] | RW | 1'b1 | bfm_soft_rst_n：1=释放，0=复位 |
| [31:5] | RSVD | 27'h0 | 保留 |

软件复位流程：
1. 向对应位写 0 → 模块进入复位状态
2. 保持至少 2 个 hclk 周期
3. 向对应位写 1 → 模块释放复位

### 9.3 RMU_STATUS（0x04）— 复位状态寄存器

| 位 | 属性 | 复位值 | 说明 |
|----|------|--------|------|
| [0] | RO | 1'b0 | pin_rst_n 滤波后状态：1=正常，0=复位中 |
| [1] | RO | 1'b0 | por_rst_n 状态：1=正常，0=复位中 |
| [2] | RO | 1'b0 | 滤波计数进行中标志 |
| [31:3] | RSVD | 29'h0 | 保留 |

## 10 软件配置流程

```
配置示例1：软件复位 UART 模块
  1. 写 RMU_SRST[0] = 0     // 拉低 UART 软复位
  2. 等待至少 2 个 hclk 周期
  3. 写 RMU_SRST[0] = 1     // 释放 UART 复位

配置示例2：查询当前复位状态
  1. 读 RMU_STATUS
     - bit[0]=1  → pin_rst_n 正常（已滤波，高电平）
     - bit[1]=1  → por_rst_n 正常（已释放）
     - bit[2]=1  → 滤波计数器工作中
```
